Reparación de Discos y USB

Herramientas profesionales para diagnosticar fallos físicos (Bad Sectors) y recuperar datos de unidades moribundas.

1. Diagnóstico de Salud (S.M.A.R.T.)

Antes de intentar reparar, pregunta al disco cómo se siente. Si falla S.M.A.R.T., haz backup y reemplázalo.

SMARTCTL (smartmontools)
$sudo smartctl -a /dev/sda
# Busca "PASSED" o "FAILED" y mira "Reallocated_Sector_Ct"

$sudo smartctl -t short /dev/sda # Ejecuta un test rápido (unos minutos)

2. Reparar Sistema de Archivos (FSCK)

Para errores lógicos (carpetas vacías, archivos corruptos). NUNCA usar en discos montados.

$umount /dev/sdb1 # Desmontar primero

$sudo fsck -y /dev/sdb1 # -y: Reparar todo automáticamente (Yes a todo)
💀 Recuperar de Sectores Defectuosos (Bad Clusters) +

Si el disco tiene daño físico (ruidos, lentitud extrema), NO uses dd. Usa ddrescue.

DDRESCUE (El salvavidas)
# 1. Instalar: sudo apt install gddrescue

$sudo ddrescue -n /dev/dañado imagen.img mapa.log # -n: No intentar scrapear (modo rápido para sacar lo fácil primero)

$sudo ddrescue -r3 /dev/dañado imagen.img mapa.log # -r3: Reintentar sectores malos 3 veces (Solo después del paso 1)

Advertencia: ddrescue clona el disco bit a bit. Necesitas otro disco con IGUAL o MAYOR espacio.

💾 Revivir USB (Formateo Profundo) +

Si un USB no se monta o da errores raros, límpialo de raíz.

Opción A: Wipe (Peligroso)
$sudo dd if=/dev/zero of=/dev/sdx bs=4M status=progress # Llena el USB de ceros. Adios virus, adios datos, adios particiones.
Opción B: Crear Nuevo Sistema
$sudo mkfs.vfat /dev/sdx1 # Formato compatible (FAT32)
$sudo mkfs.ext4 /dev/sdx1 # Formato Linux

Personalizar Iconos de Apps

¿No te gusta el icono de VS Code o Chrome? Cámbialo. En Linux (Ubuntu/GNOME), esto se hace editando los archivos .desktop.

La Teoría: Archivos .desktop

En Linux, los accesos directos no son .exe ni .lnk. Son archivos de texto con extensión .desktop. Contienen el nombre, el comando a ejecutar y la ruta del icono.

Ubicaciones Clave

  • /usr/share/applications/: Apps del sistema (Para todos los usuarios). Requiere Root.
  • ~/.local/share/applications/: Apps de tu usuario. (Recomendado editar aquí).
🎨 Guía Paso a Paso (Método Manual) +

Vamos a cambiar el icono de una app (ej: Firefox) sin romper el sistema.

1. Consigue tu icono

Busca una imagen PNG o SVG. Guárdala en una carpeta segura, por ejemplo:

$mkdir -p ~/.local/share/icons
$cp mi_icono_cool.png ~/.local/share/icons/

2. Copia el .desktop a tu carpeta local

Nunca edites los archivos en /usr/share directamente. Cópialo a tu carpeta personal; el sistema le dará prioridad.

$cp /usr/share/applications/firefox.desktop ~/.local/share/applications/

3. Edita la línea "Icon"

$nano ~/.local/share/applications/firefox.desktop

# Busca la línea que empieza por 'Icon=' y cámbiala:
Icon=/home/john/.local/share/icons/mi_icono_cool.png

# Guarda con Ctrl+O y sal con Ctrl+X

4. Refrescar

A veces los cambios son inmediatos. Si no:

  • Presiona Alt + F2, escribe r y pulsa Enter (Solo en X11/GNOME).
  • O cierra sesión y vuelve a entrar.

Alternativa Gráfica: MenuLibre

Si prefieres no usar la terminal, existe una app genial para esto.

$sudo apt install menulibre
# Luego abre "Editor de Menús" en tus aplicaciones.

Depuración y Mantenimiento

Comandos sagrados para cuando "algo se rompe" o "el disco está lleno pero no veo archivos".

1. Archivos Fantasma (Espacio Disco)

¿Borraste un archivo gigante pero df -h dice que el disco sigue lleno? Un proceso lo tiene "agarrado".

$lsof +L1 # Lista archivos con 0 enlaces (borrados) pero abiertos.

$kill -9 PID # Mata el proceso (PID) para liberar el espacio realmente.
🔪 Matar Procesos (Kill / Zombies) +

Si una app se cuelga, no reinicies. Mátala.

Identificar
$top # O htop. Busca procesos en estado 'Z' (Zombie).
$ps aux | grep firefox
Ejecutar
$kill 1234 # Pide amablemente terminar (SIGTERM)
$kill -9 1234 # Fuerza bruta (SIGKILL). Úsalo si no responde.
$pkill -f chrome # Mata todo lo que se llame "chrome".

2. Reparar Actualizaciones (APT/DPKG)

Si se fue la luz actualizando o ves "Errors were encountered while processing...", usa esto.

# 1. Reconfigurar paquetes a medio instalar
$sudo dpkg --configure -a

# 2. Arreglar dependencias rotas
$sudo apt --fix-broken install

# 3. Limpiar caché corrupta
$sudo apt clean && sudo apt update
♻️ Recuperar Archivos Borrados +

Si borraste algo con rm, no va a la papelera. Es difícil volver, pero herramientas como TestDisk pueden ayudar.

$sudo apt install testdisk
$sudo testdisk # Sigue el asistente: [Create] -> Select Disk -> [Intel] -> [Advanced] -> [Undelete]

Nota: Para recuperación forense de fotos, usa photorec (viene con testdisk).

3. Auditoría: Grabar Historial

Para ver qué hiciste ayer o grabar una sesión completa para mostrársela a un experto.

Exportar Historial Completo

$history > mis_comandos.txt

# 2. Leer TODO el historial histórico (archivo oculto):
$cat ~/.bash_history > historial_completo.txt

Graba todo lo que sale por pantalla (inputs y outputs) hasta que escribas exit.

$script auditoria.log
# ... escribe tus comandos peligrosos ...
$exit
# El archivo 'auditoria.log' tendrás todo grabado.

Desinstalación de Paquetes y Depuración

Guía completa para desinstalar programas de forma eficiente, limpiar dependencias huérfanas y liberar espacio en Ubuntu/Debian.

⚠️ Remove vs Purge: Diferencias Clave

apt remove

  • Desinstala el programa
  • ❌ Deja archivos de configuración
  • ✅ Si reinstalas, conserva tu configuración
  • Usa cuando: Vas a reinstalar después

apt purge

  • Desinstala el programa
  • ✅ Borra archivos de configuración (/etc)
  • ✅ Limpieza completa del sistema
  • Usa cuando: No volverás a usar ese programa

1. Desinstalación Básica (APT)

REMOVE - Desinstalar conservando configuración
# Desinstalar un paquete (mantiene configuración):
$sudo apt remove firefox

# Desinstalar múltiples paquetes:
$sudo apt remove firefox vlc gimp

# Simulación (ver qué se va a borrar sin hacerlo):
$sudo apt remove --dry-run firefox
# Útil para verificar antes de borrar
PURGE - Desinstalación completa (incluye configuración)
# Desinstalar + borrar configuración:
$sudo apt purge firefox

# Purgar múltiples paquetes a la vez:
$sudo apt purge firefox vlc gimp

# Purgar un paquete ya removido (solo borra configs):
$sudo apt purge firefox # Funciona aunque ya esté desinstalado

2. Limpieza Profunda: Dependencias Huérfanas

Cuando instalas un programa, trae dependencias (librerías). Al desinstalarlo, esas dependencias quedan "huérfanas" ocupando espacio.

AUTOREMOVE - Eliminar dependencias no usadas
# Después de desinstalar, limpia dependencias huérfanas:
$sudo apt autoremove

# Autoremove + purge (borra configs de las dependencias):
$sudo apt autoremove --purge

# Comando recomendado para máxima limpieza:
$sudo apt purge firefox && sudo apt autoremove --purge

3. Limpieza de Caché de Paquetes (.deb)

APT guarda todos los .deb descargados en /var/cache/apt/archives/. Con el tiempo puede ocupar varios GB.

CLEAN vs AUTOCLEAN
# Ver cuánto espacio ocupa la caché:
$du -sh /var/cache/apt/archives/

# CLEAN - Borrar TODA la caché (agresivo):
$sudo apt clean # Libera espacio, pero tendrás que re-descargar si reinstalas

# AUTOCLEAN - Borrar solo paquetes obsoletos (recomendado):
$sudo apt autoclean # Borra versiones viejas, mantiene las actuales
🔧 DPKG - Gestión de bajo nivel +

DPKG es el gestor de paquetes base de Debian/Ubuntu. APT es una interfaz sobre dpkg.

Listar paquetes instalados
# Ver todos los paquetes instalados:
$dpkg -l

# Buscar un paquete específico:
$dpkg -l | grep firefox

# Ver archivos instalados por un paquete:
$dpkg -L firefox
Desinstalar con DPKG
# Remover paquete (similar a apt remove):
$sudo dpkg -r firefox

# Purgar paquete (similar a apt purge):
$sudo dpkg -P firefox

# ⚠️ DPKG no resuelve dependencias automáticamente
# Si falla, luego ejecuta: sudo apt --fix-broken install
Encontrar paquetes con configuración residual
# Listar paquetes marcados como 'rc' (removed, config remains):
$dpkg -l | grep '^rc'

# Purgar TODOS los paquetes con config residual:
$sudo dpkg -l | grep '^rc' | awk '{print $2}' | xargs sudo dpkg --purge
📦 Desinstalar Snap y Flatpak +

Ubuntu usa Snap para algunos programas. Flatpak es otra alternativa moderna.

SNAP - Paquetes universales de Canonical
# Listar snaps instalados:
$snap list

# Desinstalar un snap:
$sudo snap remove firefox

# Remover + borrar datos del usuario:
$sudo snap remove --purge firefox

# Deshabilitar snap completamente (avanzado):
$sudo systemctl disable snapd
$sudo apt purge snapd
FLATPAK - Paquetes de Flathub
# Listar flatpaks instalados:
$flatpak list

# Desinstalar flatpak:
$flatpak uninstall org.mozilla.firefox

# Borrar datos del usuario también:
$flatpak uninstall --delete-data org.mozilla.firefox

# Limpiar runtimes no usados (equivalente a autoremove):
$flatpak uninstall --unused

🚀 Rutina de Limpieza Completa (Recomendada)

Ejecuta esto mensualmente para mantener tu sistema optimizado:

# 1. Actualizar lista de paquetes
$sudo apt update

# 2. Actualizar paquetes instalados
$sudo apt upgrade -y

# 3. Eliminar paquetes huérfanos + configs
$sudo apt autoremove --purge -y

# 4. Limpiar caché de paquetes obsoletos
$sudo apt autoclean

# 5. Purgar configuraciones residuales
$sudo dpkg -l | grep '^rc' | awk '{print $2}' | xargs -r sudo dpkg --purge

# 6. Limpiar journald logs (opcional, si /var está lleno)
$sudo journalctl --vacuum-time=7d # Mantiene solo últimos 7 días

# 7. Ver espacio liberado
$df -h
🔌 Desinstalar PPAs (Repositorios Externos) +

Los PPAs (Personal Package Archives) añaden software de terceros. A veces conviene eliminarlos.

# Listar PPAs instalados:
$ls /etc/apt/sources.list.d/

# Remover un PPA:
$sudo add-apt-repository --remove ppa:nombre/repo

# Desinstalar programa del PPA + remover el repositorio:
$sudo apt purge nombre-paquete
$sudo add-apt-repository --remove ppa:nombre/repo
$sudo apt update

⚠️ Zona Peligrosa: Comandos Avanzados

Usar con EXTREMA precaución. Pueden romper tu sistema.

Forzar desinstalación (si dpkg/apt fallan)
# Si un paquete está roto y no se puede desinstalar normalmente:
$sudo dpkg --force-all -P nombre-paquete

# Luego arregla dependencias:
$sudo apt --fix-broken install
Remover kernels viejos (liberar espacio /boot)
# Ver kernels instalados:
$dpkg -l | grep linux-image

# Ver kernel actual (NO LO BORRES):
$uname -r

# Remover kernel viejo específico:
$sudo apt purge linux-image-5.15.0-56-generic

# Autoremover kernels viejos (más seguro):
$sudo apt autoremove --purge

⚠️ Recomendaciones Críticas para 2026

Para garantizar la estabilidad del sistema operativo en Ubuntu 24.04 LTS y sucesoras:

1. Uso Obligatorio de Entornos Virtuales

En 2026, Ubuntu aplica estrictamente la política de Externally Managed Environments.

# ❌ NO USAR (romperá el sistema):
$sudo pip install paquete

# ✅ CORRECTO - Crear entorno virtual:
$python3 -m venv mi_entorno

# Activar entorno:
$source mi_entorno/bin/activate

# Ahora sí puedes instalar con pip:
(mi_entorno) $pip install requests pandas numpy

# Desactivar entorno cuando termines:
(mi_entorno) $deactivate

2. Seguridad con el PPA Deadsnakes

El repositorio Deadsnakes es la fuente más confiable para obtener las últimas actualizaciones de Python sin conflictos con las dependencias nativas del sistema.

  • ✅ Mantenido activamente por la comunidad Python
  • ✅ Versiones probadas y estables para Ubuntu
  • ✅ No interfiere con el Python del sistema
  • ✅ Actualizaciones de seguridad regulares

3. Precaución con la Compilación Manual

Al instalar Python desde el código fuente:

# ✅ CORRECTO - Usar altinstall (no sobrescribe binarios del sistema):
$sudo make altinstall

# ❌ PELIGROSO - NUNCA usar make install:
# sudo make install # ¡Esto sobrescribirá /usr/bin/python!
# Ubuntu usa Python para procesos críticos del sistema

Instalar Manim (Math Animation Engine)

Manim es una librería de Python para generar animaciones matemáticas en video. Aquí tienes el flujo completo: dependencias del sistema, entorno virtual, instalación del paquete y renderizado de prueba.

1. Dependencias del Sistema

Necesitas FFmpeg, LaTeX y otros paquetes para que los renders salgan con audio/texto.

$sudo apt update
$sudo apt install ffmpeg texlive-full python3-pip python3-venv

Aíslalo para que las dependencias de Manim no choquen con otros proyectos.

Carpeta del Proyecto
$mkdir ~/manim-projects
$cd ~/manim-projects
Virtualenv
$python3 -m venv .venv

3. Instalar Manim

Usamos pip para instalar la versión estable.

pip
# Asegúrate de estar dentro de la venv

Archivo Example.py

$manim -pqh example.py Hello

Actualizar Manim
$pip install -U manim

Manejo de UV en Python

uv es el gestor de entornos y dependencias ultrarrápido de Astral. Los comandos a continuación se basan en la guía oficial (docs.astral.sh) para instalarlo y aprovecharlo en proyectos reales.

1. Instalación vía Script Oficial

Método recomendado para Linux/macOS (detecta la arquitectura y ubica el binario en ~/.local/bin).

Script (curl + sh)

Verificar versión
$uv version

2. Instalación vía Gestores

Alternativas sugeridas cuando prefieres paquetes manejados por el sistema.

Homebrew (macOS/Linux)
$brew install astral-sh/uv/uv
pipx (aislado)
$pipx install uv
Actualización
# Script:
$uv self update

# Homebrew:
$brew upgrade uv

3. Crear y Gestionar Proyectos

uv combina la creación de entornos con un lockfile reproducible.

Inicializar proyecto
$mkdir mi-app && cd mi-app

Bash

Uso de Bash

Comandos y técnicas prácticas para manipular el sistema directamente desde la terminal con Bash. Clasificados por categoría para facilitar el aprendizaje.

1. Variables y Entorno

Declarar, exportar y consultar variables de entorno en Bash.

Variables y entorno
# Declarar una variable:
$nombre="Linux"

# Usar la variable:
$echo "Hola $nombre"

# Exportar variable al entorno (disponible en subprocesos):
$export MI_VAR="valor"

# Ver todas las variables de entorno:
$printenv

# Ver el valor de una variable específica:
$echo $HOME $USER $SHELL

# Eliminar una variable:
$unset MI_VAR

2. Manipulación de Cadenas

Operaciones sobre texto directamente con Bash, sin herramientas externas.

Operaciones con cadenas
# Longitud de una cadena:
$texto="hola mundo"
$echo ${#texto} # → 10

# Subcadena (desde posición 5, 5 caracteres):
$echo ${texto:5:5} # → mundo

# Reemplazar primera ocurrencia:
$echo ${texto/hola/adiós} # → adiós mundo

# Convertir a mayúsculas / minúsculas:
$echo ${texto^^} # → HOLA MUNDO
$echo ${texto,,} # → hola mundo

3. Condicionales (if / case)

Tomar decisiones en Bash según el estado del sistema o valores de variables.

if / elif / else
# Verificar si un archivo existe:
$if [ -f "archivo.txt" ]; then
     echo "El archivo existe"
elif [ -d "archivo.txt" ]; then
     echo "Es un directorio"
else
     echo "No existe"
fi

# Comparar números (-eq -ne -lt -gt -le -ge):
$if [ $num -gt 10 ]; then echo "mayor"; fi
case
$case $1 in
  start) echo "Iniciando..." ;;
  stop) echo "Deteniendo..." ;;
  *) echo "Uso: start|stop" ;;
esac

4. Bucles (for / while / until)

Iterar sobre listas, rangos o condiciones para automatizar tareas repetitivas.

for — iterar sobre lista y rango
# Iterar sobre una lista de valores:
$for fruta in manzana pera uva; do
    echo "Fruta: $fruta"
done

# Iterar sobre un rango numérico:
$for i in {1..5}; do echo $i; done

# Iterar sobre archivos de un directorio:
$for f in ~/Documentos/*.txt; do
    echo "Procesando: $f"
done
while / until
# while: ejecuta mientras la condición sea verdadera:
$contador=0
$while [ $contador -lt 5 ]; do
    echo "Vuelta $contador"
    ((contador++))
done

# until: ejecuta hasta que la condición sea verdadera:
$until [ $contador -ge 5 ]; do echo $contador; ((contador++)); done

5. Arrays (Arreglos)

Almacenar y manipular listas de elementos en una sola variable.

Declarar y acceder a arrays
# Declarar un array:
$colores=("rojo" "verde" "azul")

# Acceder a un elemento (índice base 0):
$echo ${colores[0]} # → rojo

# Todos los elementos:
$echo ${colores[@]}

# Número de elementos:
$echo ${#colores[@]} # → 3

# Agregar un elemento al array:
$colores+=("amarillo")

# Iterar sobre el array:
$for c in "${colores[@]}"; do echo $c; done

6. Funciones

Reutilizar bloques de código y pasar argumentos dentro de un script Bash.

Definir y llamar funciones
# Definir una función:
$saludar() {
    echo "Hola, $1!" # $1 = primer argumento
}

# Llamar la función:
$saludar mundo # → Hola, mundo!

# Función con valor de retorno (via variable global):
$sumar() { resultado=$(( $1 + $2 )); }
$sumar 3 7
$echo $resultado # → 10

7. Redirección de Entrada/Salida

Controlar hacia dónde van los datos de entrada, salida y errores.

Operadores de redirección
# Redirigir stdout a un archivo (sobrescribe):
$ls -la > listado.txt

# Redirigir stdout a un archivo (agrega al final):
$echo "nueva línea" >> listado.txt

# Redirigir stderr a archivo:
$comando 2> errores.log

# Redirigir stdout y stderr juntos:
$comando > salida.log 2>&1

# Leer entrada desde archivo:
$sort < listado.txt

Crítico

Recuperación del Sistema (Boot Repair)

Si tu Linux no arranca y ves una pantalla negra con grub rescue> o un mensaje de error, no entres en pánico. Aquí tienes cómo solucionarlo.

Caso A: El prompt "grub rescue>"

Esto ocurre cuando GRUB se ha corrompido o movido. Estás en una mini-shell de emergencia.

Paso 1: Localizar tu partición Linux

grub rescue>ls # Salida: (hd0) (hd0,msdos1) (hd0,msdos1)...

# Prueba hacer 'ls' en cada una hasta ver carpetas de linux (etc, boot, home)
grub rescue>ls (hd0,msdos1)/ # Si ves "error: unknown filesystem", esa NO es.

Paso 2: Configurar y Arrancar

Supongamos que tu Linux está en (hd0,msdos1):

grub rescue>set root=(hd0,msdos1)
grub rescue>set prefix=(hd0,msdos1)/boot/grub
grub rescue>insmod normal
grub rescue>normal

Si esto funciona, tu PC arrancará. IMPORTANTE: Al entrar a Linux, abre una terminal y ejecuta sudo update-grub y sudo grub-install /dev/sdX para hacerlo permanente.

Caso B: Método "Live USB" (Chroot)

Si el método anterior falla, necesitas un USB de instalación (Ubuntu, Fedora, etc.).

🛠️ Guía Paso a Paso (Reinstalar GRUB) +

1. Arranca desde el USB ("Try Ubuntu").

2. Abre una terminal (Ctrl+Alt+T) y averigua tu partición:

$lsblk # Busca la partición grande (ej: /dev/sda2)

3. Monta tu sistema en /mnt:

$sudo mount /dev/sda2 /mnt # Si tienes partición EFI separada (boot), móntala también:
$sudo mount /dev/sda1 /mnt/boot/efi

4. Reinstala GRUB:

# Opción rápida (Boot Repair automática):
$sudo apt-add-repository ppa:yannubuntu/boot-repair
$sudo apt update && sudo apt install boot-repair
$boot-repair

# Opción Manual (Pro):
$sudo grub-install --boot-directory=/mnt/boot /dev/sda
Fundamentos

¿Qué es Linux Realmente?

Linux no es solo un sistema operativo; es el motor que impulsa internet, supercomputadoras, dispositivos Android y, cada vez más, el desarrollo de software moderno.

El Kernel

Es el "corazón". Gestiona la CPU, la memoria y los dispositivos. Fue creado por Linus Torvalds en 1991. Cuando dices "Linux", técnicamente hablas del Kernel.

GNU/Linux

El Kernel solo no sirve de mucho. Necesita herramientas (compiladores, shell, utilidades). Estas vinieron del proyecto GNU. Juntos, forman el sistema operativo completo.

Filosofía Open Source

A diferencia de Windows o macOS, Linux es código abierto. Cualquiera puede ver cómo está hecho, modificarlo y redistribuirlo. Esto ha creado una comunidad global masiva que lo mejora constantemente.

Distribuciones (Distros)

Como el Kernel de Linux es libre, muchas empresas y comunidades crean sus propias versiones empaquetadas. Estas se llaman "Distros".

  • Debian / Ubuntu: Las más populares para principiantes y servidores. Muy estables.
  • Red Hat / CentOS / Fedora: Enfocadas en el mundo empresarial corporativo.
  • Arch Linux: Para usuarios avanzados que quieren construir su sistema desde cero ("Do It Yourself").
Vital

La Jerarquía de Archivos

En Windows tienes C:\, D:\, etc. En Linux, todo empieza en un solo lugar: La Raíz (The Root), representada por una barra /.

El Árbol de Directorios

/ RAÍZ del sistema (The Root).
/bin Binarios (programas esenciales como ls, cp).
/etc Configuración del sistema (red, usuarios, wifi).
/home Carpeta personal de usuarios (Tus documentos).
/john Tu carpeta personal (~).
/var Archivos variables (Logs del sistema, webs).
/tmp Archivos temporales (se borran al reiniciar).
/dev Dispositivos (discos duros, terminales).
/proc Información del Kernel (CPU, Memoria).
